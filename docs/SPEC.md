# 📋 SPEC.md — 个人效率工具箱 功能规格

> 本文档定义每个工具的具体行为、输入输出、边界情况、UI 要求和端侧分布。
> 是写代码时对照的"最终规格"，不做模糊设计。

## 符号说明

- **✅ 纯前端** — 完全在浏览器/Flutter 本地执行，不需要网络
- **🔷 需后端** — 需要服务端资源（大文件/库不支持前端/Python 生态）

## 一、格式化 & 转换类

### 1.1 JSON 工具

**核心能力：**
- 格式化（带缩进，可选 2/4 空格或 Tab）
- 压缩（去空格换行，一行输出）
- 合法性校验（错误时定位到具体行/列）
- 树形展示（可折叠/展开节点，复制节点路径）

**输入：**
- 文本输入框（支持大文本，不卡顿）
- 示例按钮「点我填入示例 JSON」

**输出：**
- 格式化/压缩结果 + 字符数/行数统计
- 校验失败时：红色错误提示 + 行号/列号 + 附近上下文

**边界情况：**
- 空输入 → 提示"请输入 JSON"
- 非法字符 → 报错位置 + snippet
- 超大 JSON（>1MB）→ 警告 + 仍然尝试处理，但提示可能卡顿
- 空对象/数组 → 正常工作
- 深层嵌套（>10 层）→ 树形视图默认折叠到 3 层

**额外功能：** 📌 V2+
- JSON 对比（Diff 两个 JSON）
- JSON → TypeScript 类型定义生成
- JSON → 表格视图（扁平化展开）

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 1.2 Base64 工具

**核心能力：**
- 文本 ↔ Base64 编解码
- 文件 ↔ Base64 编解码（拖拽或选择文件，预览大小）
- 实时预览：编码/解码结果即时更新

**编码输入：**
- 文本输入（原始字符串 → Base64）
- 文件选择/image 拖拽（图片 → Base64 Data URI）

**解码输入：**
- Base64 字符串 → 原文
- Base64 图片 Data URI → 图片预览 + 下载为文件

**边界情况：**
- 空输入 → 输出空字符串
- 非法 Base64 字符串（含不可解码字符）→ 提示错误
- 超大文件编码（>50MB）→ 警告 + 建议用专业工具
- Unicode 字符（中文/Emoji）→ 正确处理 UTF-8

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 1.3 URL 编解码

**核心能力：**
- `encodeURIComponent` / `decodeURIComponent` — 编码整个 URL 参数
- `encodeURI` / `decodeURI` — 编码完整 URL（保留 `://` `/` `?` `#`）
- 实时输入输出

**输入/输出：** 单行/多行文本框

**边界情况：**
- 空输入 → 输出空字符串
- 混合编码（部分编码的 URL）→ 智能识别，优先尝试解码
- 非法编码序列 → 原样返回 + 警告
- 中文字符 → 正确处理

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 1.4 时间戳工具

**核心能力：**
- 秒级/毫秒级时间戳 → 可读日期时间
- 日期时间 → 秒级/毫秒级时间戳
- 多种格式输出：
  - ISO 8601: `2024-01-15T10:30:00+08:00`
  - 中国常用: `2024-01-15 10:30:00`
  - 美国格式: `01/15/2024 10:30:00 AM`
  - 相对时间: `3 小时前` / `后天` / `下周一`
- 实时预览（输入任意一端，另一端即时更新）

**输入：**
- 时间戳输入框（自动识别秒/毫秒）
- 日期时间选择器（DatePicker）
- 快速填入「当前时间」「当前时间戳」

**边界情况：**
- 非法时间戳（负数/非数字）→ 提示
- 超出范围（Unix epoch 范围外）→ 提示
- 空输入 → 展示当前时间
- 时区校正 → 默认显示 UTC+8，可选切换时区

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 1.5 颜色工具

**核心能力：**
- 多格式互转：HEX ↔ RGB ↔ HSL ↔ HSV ↔ 颜色名称
- 调色板：可拖拽选择颜色（色相环 + 明度/饱和度滑块）
- 取色器：从图片上传中取色（点击图片某点得颜色值）📌 V2+
- 颜色预览：大色块展示 + 深色/浅色文字对比

**输入：**
- 任意格式颜色值输入
- 颜色选择器交互

**输出：**
- 所有格式的转换结果 + 色块展示
- 最近 10 个取色历史 📌 V2+

**边界情况：**
- 简写 HEX (`#fff`) → 展开为 `#ffffff`
- 无 `#` 前缀 → 自动补全
- 非法颜色值 → 红色提示 + 无预览
- RGBA/HSLA 带透明度 → 支持 alpha 通道

**端侧：** ✅ 纯前端 · 🧩 可离线

## 二、编码 & 加密类

### 2.1 UUID / NanoID 生成器

**核心能力：**
- 批量生成（1-100 个，可选数量）
- UUID 版本：v4（随机） / v7（按时间排序）📌 V2+
- NanoID 生成：可自定义长度和字符集
- 格式选项：大写/小写/去横线/带花括号

**输入：**
- 数量选择器
- 版本选择
- 格式定制

**输出：**
- 列表展示
- 一键复制全部 / 逐条复制

**边界情况：**
- 生成 0 个或负数 → 默认 1 个
- 超过 100 个 → 提示"最多 100 个"

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 2.2 哈希工具

**核心能力：**
- 支持的算法：
  - MD5 (128bit)
  - SHA-1 (160bit)
  - SHA-256 / SHA-224
  - SHA-512 / SHA-384
  - SHA-3 (256/512) 📌 V2+
- 输入方式：文本输入 / 文件输入 / 拖拽

**输出：**
- 所有选定算法的哈希值（十六进制小写/大写可选）
- 哈希值比对：两段文本/文件的哈希是否一致 📌 V2+

**边界情况：**
- 空输入 → 算法输出 "" 的哈希（标准值）
- 大文件哈希 → 分块读取，前端用 Web Worker / Flutter isolate
- 超大文本（>100KB）→ 自动切换到文件模式

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 2.3 JWT 解码器

**核心能力：**
- 解析 JWT token → 展示 Header / Payload / Signature
- Payload 美化展示（格式化 JSON 树）
- 自动检测过期时间 → 剩余有效时间 / 是否已过期
- 一键复制各段

**输入：**
- Token 粘贴框（自动识别 JWT 格式）

**输出：**
- Header 解码
- Payload 解码 + 注册声明标注（iss/sub/iat/exp 等）
- Signature（十六进制 + 原始 base64）
- 有效期状态条

**边界情况：**
- 非法 token 格式（非三段式）→ 提示
- 非 base64url 编码 → 尝试修复
- 过期 token → 红色警告 + 过期时间
- 未过期 → 绿色剩余时间

**端侧：** ✅ 纯前端 · 🧩 可离线

## 三、文本处理类

### 3.1 文本统计

**核心能力：**
- 统计：
  - 字符数（含空格 / 不含空格）
  - 单词数
  - 行数
  - 中文字数
  - 字节数（UTF-8 / UTF-16）
  - 数字/字母/标点/空格 各自数量
- 实时统计（输入即更新）

**输入：** 多行文本框

**输出：** 卡片式统计面板

**边界情况：**
- 空输入 → 全部归零
- 纯空格/空行 → 正确计数
- 混合中英文/Emoji → 正确计算（Emoji 可能占 2 个字符）
- 超大文本（>100KB）→ 仍然实时，但降低更新频率

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 3.2 文字格式互转

**核心能力：**
- 大小写转换：大写 / 小写 / 首字母大写（Title Case） / 句子首字母大写（Sentence case）
- 命名法互转：
  - camelCase
  - PascalCase
  - snake_case
  - kebab-case
  - 中划线命名法
  - 空格分隔
- 反转字符串

**输入/输出：** 文本框 + 各格式预览结果

**边界情况：**
- 空输入 → 所有输出为空
- 数字开头 → camelCase / PascalCase 逻辑（数字保留）
- 特殊字符 → 按分隔符智能拆分
- 中文/混合文案 → 保留原样，只处理英文部分

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 3.3 正则表达式测试器

**核心能力：**
- 输入正则 + 测试文本 → 实时匹配高亮
- 匹配结果列表（每个 match 的起始位置 + 内容）
- Flags 开关：`g` `i` `m` `s` `u` `y`
- 正则语法提示（常用模式速查表）
- 替换测试：输入替换文本，预览替换结果 📌 V2+

**输入：**
- 正则输入框（语法高亮）📌 V2+
- 测试文本多行输入
- Flags 开关按钮

**输出：**
- 文本中匹配片段高亮
- 匹配结果列表（序号 / 匹配内容 / 位置）
- 替换结果预览（V2+）

**边界情况：**
- 非法正则 → 实时报错（显示具体错误位置）
- 空正则 → 空匹配（或提示输入正则）
- 大量匹配（>1000 个）→ 限制显示数量 + 提示
- 回溯爆炸风险 → 检测超时，自动切断 + 提示优化正则

**端侧：** ✅ 纯前端 · 🧩 可离线

## 四、开发者工具

### 4.1 HTML 实体编解码

**核心能力：**
- 编码：`<` → `&lt;` 等 5 个预定义实体 + 十进制/十六进制编码
- 解码：实体 → 原始字符
- 支持：HTML4 命名实体 / 十进制 / 十六进制

**输入/输出：** 文本框

**边界情况：**
- 空输入 → 空输出
- 混合编码/未编码文本 → 只处理匹配的部分
- 未知命名实体 → 保留原样

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 4.2 Unicode 码点查询

**核心能力：**
- 输入字符 → 显示 Unicode 码点（U+XXXX）
- 输入码点 → 显示对应字符
- 显示基本信息：所属区块、名称（如 "CJK Unified Ideograph"）
- 批量输入：一段文本 → 列出每个字符的码点

**边界情况：**
- Emoji 序列（多码点组合）→ 解析每个码点 + 组合关系说明
- 非法码点（U+110000+）→ 提示
- 控制字符（U+0000-U+001F）→ 显示名称而不是不可见字符

**端侧：** ✅ 纯前端 · 🧩 可离线

---

### 4.3 Cron 表达式解析

**核心能力：**
- 输入 Cron 表达式 → 解析为人类可读描述
- 预览接下来 5/10/20 次执行时间
- 支持标准 5 字段 + 6 字段（秒）
- 常用预设：`每5分钟` `每天 0 点` `每周一早 9 点` 等
- 支持缩写：`@hourly` `@daily` `@weekly` `@monthly` `@yearly`

**输入：**
- Cron 文本输入
- 快捷预设选择

**输出：**
- 人类可读描述（中文）
- 执行时间列表（可复制）
- 可视化时间轴（24h 热度图）📌 V2+

**边界情况：**
- 非法表达式 → 错误位置 + 语法提示
- `* * * * *` → "每分钟执行"
- 复杂表达式（如 `0 0 1 1 *`）→ 正确解析
- 特殊字符（L W #）→ 支持部分（V2+ 全覆盖）

**端侧：** ✅ 纯前端 · 🧩 可离线

## 八、UI/UX 规格（全局）

### 布局
- **Web**: 左侧导航（分类树）+ 右侧内容区，移动端切换为底部 Tab
- **Flutter**: NavigationRail（宽屏）+ BottomNavigationBar（窄屏），自适应
- **桌面**: 可调整宽度的侧边栏 + 内容区

### 暗色模式
- 跟随系统 + 手动切换
- 所有工具支持深色/浅色

### 快捷键
- `Cmd/Ctrl + K` → 全局搜索工具（V2+）
- `Cmd/Ctrl + Enter` → 执行/格式化
- `Cmd/Ctrl + C` → 自动复制结果（V2+ 可配）

### 响应式断点
- `< 640px` — 手机：底部 Tab，全宽内容
- `640-1024px` — 平板：折叠侧栏 + 内容区
- `> 1024px` — 桌面：固定侧栏 + 内容区
